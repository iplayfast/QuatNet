Performance comparison (insertion, search, deletion):
- Sorted list: O(n) insert, O(log n) search (binary search), O(n) delete
- Unbalanced BST: O(n) worst case (degenerate to linked list), O(log n) avg
- Balanced BST (AVL, Red-Black, 2-3): O(log n) all operations, guaranteed

Good input sequences for stress testing:
- Sorted ascending (worst for unbalanced BST)
- Sorted descending
- Alternating high/low
- Random insertions

For (a,b)-trees, good values depend on memory hierarchy:
- In-memory: (2,3) or (2,4) work well (2-3 tree, 2-3-4 tree)
- Disk-based: larger b (100-1000) to match block/page size
- Higher a increases fan-out, reducing tree height
- Balance parameter: (2,b) gives minimum redundancy; (b/2,b) gives symmetric split
